Java基础学习笔记之二 - 面向对象编程的基本特征

根据在Github上的开源项目《On Java 8》中文版的相关内容编制而成，原书作者为[美] Bruce Eckel ，即《Java编程思想》的作者。

封装

Java通过访问修饰符（access specifier）来限制类、成员和方法的访问控制权限，并将此分为public，protected，包访问权限（package access，无修饰符）和private，如下所示：

本类 本包 子类 所有
public Y Y Y Y
protected Y Y Y
package access, 无修饰符 Y Y
private Y

其中，普通的类只能用public 或（无）进行修饰，若想类仅为自己或子类所用，可以用private或protected修饰构造器的方式来达到此目的。

类的集合称之为包，其被组织在一个单独的命名空间（namespace）下，并用package声明和import导入（可采用通配符*可以导入整个包 ）。注意：当采用import static导入包时，在调用其包含的类的静态成员和方法可以不通过包名而直接使用。一个Java源代码文件称为一个编译单元（compilation unit），其可以包含一个或多个类，但每个编译单元中只能有一个public类，且类名必须与文件名相同。

复用

Java通过组合、继承和委托实现代码复用，如下：

  • 组合：将已有类的对象作为新类的成员即为组合；组合的初始化方式包括：1）在对象定义时或在新类构造器中赋值进行初始化；2）在对象使用前判断其值是否未null，继而赋值进行初始化；3）在代码块中（静态、非静态）进行初始化。
  • 继承：语法为accessspecifier class SubTypeName extends SuperTypeName{}；继承的特性包括：1）仅支持单继承；2）继承时将调用父类构造器，当父类构造器带参数时，子类将不会自动创建无参构造器而必须使用super(args)调用相应的父类构造器；3）使用super.fields或super.methods可以调用父类成员或方法，当子类方法与父类方法重名（未构成重写）时，可以通过参数列表的不同自动判断是父类方法还是子类方法；4）子类是父类的一种类型，子类对象可以作为父类对象直接使用（即向上转型）。
  • 委托：Java不直接支持委托，但可以借助组合以实现，即将需要委托的类声明为私有成员，并通过公开的方法调用所需要的私有成员的部分或全部方法。

多态

Java利用“向上转型”的特性可实现多态，即将子类对象的引用赋值予父类对象，此时调用父类方法将实现对应子类（重写）方法。Java中多态的实现源于方法的“后期绑定”。使用多态时应注意：1）多态针对方法，成员不构成多态；2）private，static，final方法均不构成多态；3）父类构造器内部存在子类重写的方法时，创建子类对象将调用父类构造器并执行子类重写的方法。

抽象

用abstract修饰的类称为抽象类，其可包含一个或多个抽象方法（即用abstract修饰的方法，其只有声明，但没有方法体），也可不包含抽象方法。抽象类不能用new来创建对象，但可以被继承，其子类若没有实现父类的所有抽象方法，则其仍为抽象类，不能用new来创建对象。

接口

接口用interface创建，用implements实现，其仅可包含：

  • 隐式的public abstract修饰的抽象方法；

  • default修饰的默认方法，其可以有方法体，并在其实现中可以直接通过Object.DefaultMethodName进行调用；

  • 静态方法，其可以有方法体，并可通过Interface.StaticMethodName进行调用，且可以包含main()方法；

  • 隐式的static final修饰的成员变量。

普通的接口只能使用public或（无）进行修饰。接口可继承多个接口，类也可实现多个接口，当多个接口内包含相同签名的方法时，必须在子接口或实现的类中重写该方法以确保其能够正确的执行。实现接口的类的对象可向上转型为其实现的任一接口。

在接口中可以定义接口，但必须遵循接口定义的原则；在类中也可以定义接口，且可以是private接口（仅能为本类所用）。在接口中可以定义类，但不能是private类。实现包含接口的接口或继承包含接口的类，并不需要实现其内部的接口，必要时可以选择性实现部分或全部内部接口。

内部类

定义在另一个类中的类，即为内部类。使用内部类可以把一些逻辑相关的类组织在一起。通常情况下，内部类可以分为成员内部类、静态内部类、局部内部类和匿名内部类。

  • 成员内部类：

    成员内部类的对象的创建与初始化，在外部类的非静态方法或其自身内部与普通类相同，但在其它位置必须依赖于外部类对象而非外部类。所以，成员内部类中不能包含任何static元素（但可以包含final static变量），其初始化方式：1）OuterObjectName.new InnerClassName(args)或调用在外部类中定义的返回内部类引用的方法accessspecifier ReturnType methodName(Argument List){return new InnerClassName(args)}。

    成员内部类具有很好的封装特性（可以使用private和protected进行修饰），并能够访问外部类中所有的元素（包括使用private和protected修饰的元素）。同时，在成员内部类中可以使用OuterClassName.this来返回外部类对象的引用，借此方法可以利用成员内部类对象调用外部类方法。

    每个成员内部类都能独立的继承一个类或实现接口，而与外部类无关，但其对象可以向上转型为外部类的父类或任一接口。因此，利用此特性可使外部类实现多重继承（类或抽象类）。此外，同一成员内部类可以有多个不同状态信息的实例，也可以将多个成员内部类以不同的方式实现同一个接口或继承同一个类，还可以利用不同成员内部类实现两种同名方法的调用。

  • 局部内部类

    定义于方法内部或某作用域内的类称为局部内部类，其只能在其所属方法或作用域中使用。通常情况下，使用局部内部类的原因在于：1）利用局部内部类对象的“向上转型”，当其实现某个接口或继承某个类时，便可通过其所属方法的return new InnerClassName(args)语句返回该接口或类的引用；2）利用局部内部类来辅助解决复杂问题，但又不希望其是公共的。

    局部内部类所在方法使用参数传递外部对象时，该方法参数将被隐式的声明为final，即在局部内部类中不能修改其值。

  • 匿名内部类

    将返回值的生成与表示这个返回值的类的定义相结合的局部内部类，被称为匿名内部类：

    new InterfaceName(){
// Anonymous Inner Class Definition  
};
new OuterClassName(){
  // Anonymous Inner Class Definition
};

    匿名内部类只能继承类或实现单一接口（两者取其一），没有构造器（但可通过实例初始化（代码块）来实现构造器功能），其中的方法也不能是抽象方法。因此，匿名内部类不适合于需要使用或重载构造器，或需要该类的多个对象的场合（可使用局部内部类）。

  • 静态内部类

    用static修饰的内部类称为静态内部类，静态内部类中可以包含static元素。静态内部类的对象的创建与初始化可以没有外部类对象，而直接采用new OuterClassName.InnerClassName(args)的形式。定义于接口中的内部类将隐式的被public static修饰（即为静态内部类），该内部类即可实现外部接口，也可定义main方法，通常用于接口实现的公共代码。

内部类的继承和覆盖

在继承内部类时，其构造函数必须以外部类的对象为参数，并通过该对象调用内部类的构造函数，如下：

class OuterClassName{
    class InnerClassName{}
}

public class InheritClassName extends OuterClassName.InnerClassName{
    InheritClassName(OuterClassName out){
        out.super();
    }
}

在继承外部类时，若自身的内部类与父类的内部类同名时，并不能覆盖。两者位于不同的命名空间。
