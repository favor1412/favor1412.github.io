Java基础学习笔记(一) - 基础知识

根据在Github上的开源项目《On Java 8》中文版的相关内容编制而成，原书作者为[美] Bruce Eckel ，即《Java编程思想》的作者。

类和对象的创建

在Java中万物皆对象，我们可以通过“引用”来操纵对象，对象与引用均可以单独存在。在Java中所做全部工作就是“定义类”，“产生对象”，并“发送消息给对象”，使其完成所需功能。类和对象的创建如下：

accessspecifier class ATypeName{ 
    /*
     * Class Body including Fields and Methods
    */	
} 

ATypeName objectName = new ATypeName(args); // Strings also can be initialized with quoted text, likes String s = "asdf"

在Java中成员Fields可以是任何类型的对象或基本类型，即使不对其赋值，Java也能保证其初始化（不包括其它临时变量）。其中，通用对象的初始值为null，基本类型的初始值如下：

基本类型 boolean char byte short int long float double void
存储空间 true/false 16 bits 8 bits 16 bits 32 bits 64 bits 32 bits 64 bits ——
初始值 false ‘\u0000’(null) 0 0 0 0L 0.0f 0.0d ——

除BigInteger（支持任意精度整数运算）和BigDecimal（支持任意精度定点数运算）除外，其它基本类型都对应封装类型，具体如下，相应基本类型与封装类型可用=直接转换。

基本类型 boolean char byte short int long float double void
封装类型 Boolean Character Byte Short Integer Long Float Double Void

类型之间可以相互转换，对于基本类型可以隐式的向更大范围类型实现自动转换，也可显示的使用(ClassType)进行强制转换；对于对象可以向上转换为父类对象。

对于常量可以通过在数值上增加前缀、后缀或以指数形式以区分不同的常量类型或使用_进行分隔位数较多的数值，使其显示更为清晰，如下：

类型 说明
后缀 L：long型、F：float型、D：double型；
前缀 0b：二进制、0：八进制、0x：十六进制；
指数 使用e或E，表示10的幂；
分隔 必须遵循以下原则：1）仅限于单下划线，不能多条相连；2）不能在数值的开头和结尾使用；3）不能在L、F、D之后使用；4）不能在二进制b和十六进制x的前后使用。

在Java中方法Methods只能在类中进行定义，如下：

accessspecifier ReturnType methodName（/* Argument List */）{  
    //The methodName and Argument List are called the signature of the method.
    //Method body.
}

在方法中的return语句具有两个作用：终止程序和指明返回值语句。注意：若已指明方法的返回值类型，则必须使用return以及返回值语句；若方法的返回值类型为void，则可以不使用return，或使用return但不能有返回值语句。

Fields和Methods在所定义的类中可以直接通过field或method调用；在其它位置可以通过objectName.field或objectName.method的方式进行调用；采用static修饰的静态Fields和Methods为类共用，可以通过ATypeName.field或ATypeName.method的方式进行调用。非静态可以调用静态，但静态不能调用非静态。

变量的作用域

在Java中变量的作用域用{}限制，且镶套无效（即镶套仍不允许变量重名）。采用static{}的方式可以定义静态块，静态块的执行早于非静态块（静态块仅在第一次new时执行一次，非静态块在每次new时都会执行，两者的执行均在构造函数之前）。

对象的初始化

Java初始化的顺序：

（1）父类静态变量；
（2）父类静态代码块；
（3）子类静态变量；
（4）子类静态代码块；
（5）父类普通变量；
（6）父类普通代码块；
（7）父类构造函数；
（8）子类普通变量；
（9）子类普通代码块；
（10）子类构造函数；

其中，变量初始化的顺序由定义的顺序决定，成员变量可以自动初始化，局部变量必须手动初始化；静态初始化只在必要时（需要时）发生，一旦发生将不会被再次执行；构造器是一种特殊类型的方法，其特点为：

  • 名称必须与类名完全相同，且没有返回值类型，不能使用return；

  • 可以使用accessspecifier来限制其访问，但不能被其它非访问限制的修饰符（如：final，static，abstract等）进行修饰；

  • 不能被继承，但可以被重载；

  • 构造器可以接受任何参数以实现特殊的初始化，不接受任何参数的构造器被叫做无参构造器；当不存在任何构造器时，编译器会自动创建默认的无参构造器，当已存在构造器时，则只能使用已存在的构造器进行初始化。

对象的清理

Java拥有垃圾回收器用以清理无用对象占据的内存资源（由new分配），但并不适用于其它“特殊”方式占据内存资源的释放，用户可借助finalize()方法在垃圾回收时刻做一些重要的清理工作。注意：Java的垃圾回收只与内存使用情况相关。

  • finalize()

finalize()是Object的protected方法，子类可以覆盖以实现资源清理工作，GC在回收对象之前调用该方法。finalize() 的调用具有不确定性，且最多被GC执行一次（不包括用户手动调用的情况），不建议用于“非内存资源”的清理。

  • 对象回收过程

对象回收为两次标记：第一次标记发生在对象不可达时，筛选是否进入F-Queue队列（即是否有必要执行finalize() 方法，执行条件为：已重写finalize()方法，且并未被虚拟机调用）；进入F-Queue队列的对象将由虚拟机自动创建低优先级的线程去执行这个对象的finalize() 方法，但不保证一定会执行结束；未进入F-Queue队列的对象将被直接回收。第二次标记为完成第一次标记且执行完finalize() 方法后对象仍不可达时进行回收，此时，对象唯一不被清理的机会是，利用finalize() 方法与任意对象建立联系。

  • System.gc()

System.gc() 为强制进行垃圾回收方法，可增加执行finalize() 方法的机会。

  • 需手动执行特殊清理任务时，应注意其清理的顺序应与初始化的顺序相反。

命名风格

  • 包：域名反写，全部小写，例如：net.mindview.package；
  • 类：每个单词首字母均大写，其余小写，例如：ClassName；
  • 常量：全部大写，例如：CONSTANT；
  • 变量及方法名：首个单词字母小写，后续单词首字母大写，其余小写，例如：valueName，methodName。

运算符

运算符能够接受一个或多个参数并生成一个新值，除了=、==、!=能够操作所有对象外，其它的运算符只能操作基本类型。运算符具有优先级，其决定了一个表达式各部分的运算顺序，通过括号可以明确或改变运算顺序。

类型 说明
赋值 赋值=；注意：基本类型直接赋予其值，通用对象赋予其引用。
算术 加+、减-、乘*、除/、取模%； 注意：算术运算支持与赋值同时进行的简化符，例如：x+=4；算术运算支持自加与自减，例如：++i、–i、i++、i– 。
关系 小于<、大于>、小于等于<=、大于等于>=、等于==、不等于!=；注意：对对象使用等于或不等于时，所比较的是其引用，若需要比较某项实际内容需要使用并改写equals()方法。
逻辑 与&&、或`
位运算 位与&、位或`
三元运算符 boolean-Exp? value_1 : value_2

控制流

  • 判断
if (Boolean-Exp)
    statements;
else
    statements;
  • 循环
while (Boolean-Exp)
    statements;

do 
    statements;
while (Boolean-Exp)

for (initialization;Boolean-Exp;step) // 三组条件语句用分号隔开，且均可以为空；任何一组条件语句均可以用逗号(,)分隔，但必须保证变量类型相同，语句会独立执行
    statements;

for (DateType x : y) //Foreach语句，将每一个y值赋值于x执行
    statements;
  • 选择
switch (integral-selector){
    case integral-value1:statement;break;
    case integral-value2:statement;break; 
    case integral-value3:statement;break; 
    ......
    default:statement;
}

  • 中断

    return退出当前方法；break中断当前循环；continue 中断当前迭代，返回当前循环起点。

  • 标签

    label：仅能置于循环之前，与break和continue配合使用；使用break label：中断并跳出标签所指的循环；使用continue label：中断并由标签所指循环重新进入。

this，super，static，final关键字

this表示调用方法的对象的引用，使用this.fields或this.methods可以调用相应对象的成员和方法，在构造器中第一行（只能在第一行）使用this(args)可以调用相应的构造器；

super表示父类对象（隐式创建）的引用，使用super.fields或super.methods可以调用父类成员和方法，在子类构造器中第一行（只能在第一行）使用super(args)可以调用相应的父类构造器；

static表示为静态元素，可以在没有对象的情况下使用，其可以修饰成员、方法、代码块和内部类；

final意为不能修改，其可以修饰成员、方法、方法参数和类，具体如下：

  • 修饰成员：表示其值（在对象周期内）不能改变（必须在定义时或构造器内进行赋值，多个构造器时，每个构造器均需要赋值）；若成员变量为基本类型，则表示存储值不能改变；若成员变量为引用类型，则表示其引用不能改变（但指向的对象本身可以改变）；

  • 修饰方法：表示子类不能对其进行重写（但自身和子类均可以重载），其中，private隐式的使用了final；

  • 修饰方法参数：表示该方法中不能有任何改变该参数值的语句；

  • 修饰类：表示该类不能被继承，对于final修饰的类，其未加final修饰的成员可以改变，但其方法无论是否加final修饰，均不能被重写。
